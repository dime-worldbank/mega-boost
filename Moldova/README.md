## ETL pipeline for Moldova BOOST

[MDA_transform_load_raw_dlt.py](MDA_transform_load_raw_dlt.py) is
designed to run both on Databricks (as a Delta Live Tables pipeline)
and locally as a standalone PySpark job. See the main
[README](../README.md) for Databricks instructions. Below are
instructions for local execution.

Unlike Togo — which does everything in pandas — Moldova's transform is
real Spark on both surfaces. A tiny `dlt` shim is activated when
`DATABRICKS_RUNTIME_VERSION` is not in the environment, so the same
`@dlt.table`-decorated functions execute lazily and are materialised by
an explicit `dlt.read("mda_boost_gold")` call at the bottom of the
script.

### Requirements (without Databricks)
- Python 3.10+
- Java 11 or 17 (for Spark)
- pip
- optional: [virtualenvwrapper](https://virtualenvwrapper.readthedocs.io/)
  or [virtualenv](https://virtualenv.pypa.io/)

### Setup (without Databricks)

```
# optional: mkvirtualenv moldova_boost
pip install pyspark pandas openpyxl

# Where the raw year-range CSVs (2006-15.csv, 2016-19.csv, 2020-24.csv)
# live on your machine. Default is Moldova/_onboarding/raw_csv/ if unset.
export MICRODATA_DIR=/path/to/raw_csv

# Optional: output path for the gold Parquet. Default is
# Moldova/_onboarding/reports/local_gold.parquet.
export LOCAL_GOLD_OUT=/path/to/local_gold.parquet
```

### Run (without Databricks)

```
python MDA_transform_load_raw_dlt.py
```

Spark will start a local session, read the three bronze CSVs, apply
the per-range if-else cascade, union into `mda_boost_gold`, and write
the result as Parquet. The row count matches the filtered bronze
row count (no aggregation — one row per raw expenditure record).

### Extracting the raw CSVs first

If you don't yet have the per-range CSVs, run the pandas-only local
runner under `_onboarding/` which also reads the source workbook
directly:

```
cd _onboarding
python scripts/8_run_pipeline_local.py
```

This writes a CSV version of gold to
`_onboarding/reports/local_gold.csv` for quick inspection without
PySpark. Not a replacement for the Spark path — the DLT pipeline
remains authoritative.

### Files consumed at pipeline run-time

The Spark pipeline reads two driver CSVs that must sit alongside this
file (Databricks Repos only reliably addresses same-folder files):

- `tag_rules.csv` — SUMIFS criteria per BOOST tag code, one row per
  (code × year-range × distinct formula shape).
- `code_dictionary.csv` — BOOST tag code → econ / econ_sub / func /
  func_sub canonical labels for the cross-country aggregator.

Both are regenerated by the onboarding scripts under `_onboarding/`.
See `_onboarding/README.md` for the analysis workflow and
[`AGENTS.md`](../AGENTS.md) for the country-onboarding playbook.
