# Zimbabwe workbook analysis

Offline tooling that reverse-engineers the Zimbabwe BOOST workbook so the
DLT pipeline can translate its formulas literally. Runs locally against
`temp/Zimbabwe BOOST.xlsx` — no Spark, no Databricks.

## Layout

```
_analysis/
├── reports/       ← start here if you're reviewing
│   ├── ISSUES.md                ← formula-level issues for SME triage
│   ├── overlap_report.md        ← raw-data peer overlaps (double-count risk)
│   └── workbook_inventory.md    ← sheet-level summary
├── data/          ← machine-readable CSVs consumed by the pipeline & reports
│   ├── sheet_inventory.csv
│   ├── defined_names.csv
│   ├── formula_map.csv
│   ├── hardcoded_columns.csv
│   ├── hardcoded_cells.csv
│   ├── tag_rules.csv            ← used by ZWE_transform_load_raw_dlt.py
│   ├── code_dictionary.csv      ← used by ZWE_transform_load_raw_dlt.py
│   └── overlap_detail.csv
└── scripts/       ← generators; run in numeric order
    ├── 1_extract_workbook.py
    ├── 2_audit_hardcoded.py
    ├── 3_extract_tag_rules.py
    ├── 4_generate_issues.py
    ├── 5_build_code_dictionary.py
    └── 6_detect_overlaps.py
```

## How to regenerate everything

```bash
cd Zimbabwe/_analysis
for s in scripts/*.py; do python3 "$s"; done
```

Each script is idempotent — rerun after the workbook changes.

## What each report is for

| Report | Generated by | Inputs |
|---|---|---|
| [reports/workbook_inventory.md](reports/workbook_inventory.md) | `scripts/1_extract_workbook.py` | source `.xlsx` |
| [reports/ISSUES.md](reports/ISSUES.md) | `scripts/4_generate_issues.py` (§8 appended by `scripts/5_build_code_dictionary.py`) | `data/defined_names.csv`, `data/formula_map.csv`, `data/hardcoded_columns.csv`, `data/hardcoded_cells.csv`, `data/tag_rules.csv` |
| [reports/overlap_report.md](reports/overlap_report.md) | `scripts/6_detect_overlaps.py` | source `.xlsx`, `data/tag_rules.csv`, `data/code_dictionary.csv` |

Each report's top section lists its own provenance for quick reference.

## Design principles

- **Translate formulas literally.** If something looks wrong in the
  workbook, report it in `reports/ISSUES.md`; never silently patch.
- **Driver CSVs, not code.** `data/tag_rules.csv` and
  `data/code_dictionary.csv` carry the classification logic. Changes there
  don't require touching the DLT notebook.
- **Report-only tooling.** These scripts don't mutate the source workbook.
  When the SME has a decision, they edit the source `.xlsx` upstream and
  re-run.
